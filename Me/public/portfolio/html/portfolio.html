Winger Framework

May 2013 - Present

  • Dependency Injection model in JSON for UI package
  • Abstracted game input for gamepad and keyboard/mouse
  • Functioning threaded server-client network manager
  • Currently being used in local indie game development group
  • Developed in C# XNA

During the past summer, I have been meeting with some of my peers on a regular basis to develop games. I started developing this framework to help out with some of the lower level issues that I knew we would run into in the process. I was able to create an entire UI framework that uses dependency injection and JSON to let us create UI layouts and transitions extremely quickly and modularly. I also created the base of our networking framework. It was especially difficult to test but I was able to create a simple "game" where the four of us could join through LAN and move blocks around the screen. If you would like to use this framework, or would like to just look at the source, you can download it here.

The Ender

May 2012

  • Mobile Prototype
  • One Button Mechanic
  • Integration with my 3D particle system
  • Integration with a physics engine (© Farseer Physics Engine)
  • Developed in C# XNA

After playing Canabalt and really liking it, I thought it would be fun to make my own running game. I liked the idea of the world falling apart around you so I decided to build the game so that everything would be destroyable. I used my particle system to create all of the objects in the game so that I could blast them apart using different shaped gravity objects. The goal of the game is to stay ahead of the destruction by landing on the leading edge of the platforms you are running on. The idea being that the longer you have to run on the platform the more speed you build up. The pictures of the game don't quite do it justice as I think the feeling of being chased is lost in a still shot. Check out the video of me playing the game, or if you'd like to play it yourself you can download the source code here.

Cookin' Lists

May 2012

  • Published to iOS app store
  • Developed using Xcode

I wanted to learn more about developing for the iPhone, so I decided I would make something that I would be able to use on a daily basis. The thought was to have a todo list, grocery list, and a recipee book all in one app. During development and testing, my friends and family suggested adding the menu functionality to allow people to plan out what their meals would look like in advance as well as a timer to help during cooking. The recipes and the menus can be exported to grocery lists based on the ingredients that they are made of which lets you enter a recipe's ingredients once, and then be able to export them to any number of lists. I released it to the app store and it did not make me a millionaire, but the benefit to me was learning some of the ins and outs of app development. The app is no longer on the app store, however, because my membership expired.

WPS (Winger Particle System)

January 2012

  • Component-based
  • GPU accelerated computation
  • Easily implemented and extended
  • Dynamic particle system winforms editor
  • Developed in C# XNA

I started the WPS project in my last semester of my Senior year in college for a advanced graphics class. My interest in particle systems had evolved from a basic 2D particle system I made for a class project some time at the begining of my college career. It then became a full 2D particle system that could display tens of thousands of particles. My natural progression was to take it to 3D as well as boosting performance by using the GPU. By the end of the semester I had created a component model GPU accelerated 3D particle system as well as a visual particle effect editor using winforms. I was also able to help a few of my classmates integrate my library into their projects befor the end of the semester. Naturally the first one was someone using my particle system to create a fart cloud behind their character when they pressed a specific button, classic. If you would like to see a few of my example particle effects you can check out the video. And if you would like to use it for yourself, or just view the source code you can download it here. Typing Invaders

August 2011

  • Full menu system
  • Start to finish gameplay
  • Highscore table
  • Integrated music and sound effects (NOTE: I do not own any of the sound files in the game)
  • Developed in C# XNA

This was a game that I created for a Serious Games class in college. The goal was to teach kids how to type rythmically instead of just typing fast. I created a page management system for the UI and created some UI elements like a slider and a button. The game included music and sound effects (not my own audio) which could be turned down in an options menu as well as a full tutorial that would walk kids through the basic steps of typing. The game tracked how rythmically the player typed and at the end would give a score based on rythm, accuracy, and total words and would record the score in a persistant highscore table. You can take a look at the video of the gameplay, or download it yourself and give it a try.
